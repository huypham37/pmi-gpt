## Context & Goal
**Priority: Medium**

The Project Context **Description** field in `TestCasesNavigatorPanel` is a small raw `<textarea>` that shows unrendered markdown. Users write markdown (headings, lists, etc.) but only see plain text. The textarea is cramped (3 rows) and unsuitable for editing longer descriptions.

**Goal:** Replace the inline textarea with a compact rendered markdown preview, and allow the user to open a fullscreen overlay editor (same pattern as the chat markdown pop-over) for comfortable editing.

## Current Behavior
- `TestCasesNavigatorPanel.tsx` → `ProjectContextSection` renders a `<textarea rows={3}>` for the description
- Markdown content (e.g. `### 3.2 Network deployment`) is shown as raw text
- Save triggers on blur or via a small "Save" button when dirty

## Proposed Solution

### 1. Inline: Replace textarea with rendered markdown preview
- Replace `<textarea>` with a read-only `<Markdown mode="minimal">` component
- Truncate to 2–3 lines with `overflow-hidden` and `line-clamp`
- Show an expand/edit button (↗ icon) to open fullscreen editor
- If description is empty, show a placeholder prompt ("Click to add project description…")

### 2. Fullscreen: Editable overlay using existing components
- Reuse `FullscreenOverlayBase` (same pattern as `DocumentFormattedMarkdownOverlay` in chat)
- Embed `ShikiCodeEditor` (from `apps/electron/src/renderer/components/shiki/ShikiCodeEditor.tsx`) inside the overlay for syntax-highlighted markdown editing
- Auto-save description on overlay close via `saveProjectContextDescription` IPC call
- Support copy-to-clipboard via `FullscreenOverlayBase`'s built-in `copyContent` prop
